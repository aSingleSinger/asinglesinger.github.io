:rabbit2: 命名规则

可以是字符（英文、中文）、下划线 _ 和数字，并且第一个字符不能是数字

以下划线开头的标识符有特殊意义，一般应避免使用相似的标识符

1.4.3 命名规范

  • ==模块名==尽量短小，并且全部使用小写字母，可以使用下划线分隔多个字母。 grame_main
  • ==包名==尽量短小，并且全部使用小写字母，不推荐使用下划线。com.lh
  • ==类名==采用单词首字母大写形式（Pascal风格）。 MyClass
  • ==模块内部的类==采用 _ + Pascal 风格的类名组成，例如在 MyClass 中的内部类 _InnerMyClass
  • ==函数、类的属性和方法的命名==，全部使用小写字母，多个字母之间使用下划线分割
  • ==常量命名==时采用全部大写字母，可以使用下划线
  • 使用单下划线 _ 开头的模块遍历或函数是受保护的，在使用 from xxx import * 语句从模块中导入时，这些模块变量或函数不能被导入
  • 使用双下划线 __ 开头的实例变量或方法是类私有的
  • 以双下划线开头和结尾的是 Python 的专用标识，例如 __init()__ 标识初始化函数

2.3 元组

不同于列表，元组是一种不可变序列，没有相关的增删改的一系列操作，只可以使用索引去获取元素。

2.3.1 基本操作

==创建元组==

使用 () 定义元组，语法结构如下: 元组名=(element1, element2,...,elementN)，元素与元素之间使用英文的逗号分割，元组中只有一个元素的时候，逗号也不能省略。

tup = (1)
print(type(tup))  # <class 'int'>

6.1.4 权限控制

通过对属性或方法添加单下划线、双下划线以及首尾双下划线来实现

单下划线开头：以单下划线开头的属性或方法表示 protected 受保护的成员，这类成员被视为仅供内部使用，允许类本身和子类进行方法，但实际上它可以被外部代码访问（防君子不防小人）

双下划线开头：表示 private 私有的成员，这类成员只允许定义该属性或方法的类本身进行访问

首尾双下划线：一般表示特殊的方法

class Student:

        self.__fun2()

    def gender(self):

    @gender.setter
    def gender(self, value):
        if value not in ['男', '女']:
            print('性别有误, 已将性别设置为默认值男')
            self.__gender = '男'
        else:
            self.__gender = value

6.2 继承

  • 一个子类可以继承多个父类
  • 一个父类可以拥有多个子类
  • 一个类如果没有继承任何类，则默认继承的是 object 类
  • 继承后就会获得父类公共的和受保护的成员

继承语法：class 类名(父类1, 父类2, ..., 父类N):

        self.age = age

7. 模块

7.1 模块的简介

  • 在Python中一个后缀名为 .py 的Python文件就是一个模块
  • 模块中可以定义函数、类等
  • 模块也可以避免函数、类、变量等名称相冲突的问题
  • 模块不仅提高了代码的可维护性，同时提高了代码的可重用性
  • 在给模块命名的时候要求全部使用小写字母，多个单词之间使用下划线进行分隔
  • 如果自定义模块名称与系统内置模块名称相同，那么在导入时会优先导入自定义的模块

7.2 自定义模块

模块分为两种

  1. 系统内置模块：由开发人员编写好的模块，在安装 Python 解释器时一同安装到计算机
  2. 自定义模块：一个以 .py 结尾的文件就是一个模块，新建 Python 文件，实际上就是在新建模块

==自定义模块的作用==

  1. 一是规范代码，将功能相同的函数、类等封装到一个模块中，让代码更易于阅读
  2. 与系统内置模块一样，自定义模块可被其他模块调用，提高开发效率

7.3 模块的导入

模块编写完成就可以被其他模块进行调用并使用被调用模块中的功能

语法结构

:tiger: import 模块名称 [as 别名]

定义 info.py 模块

name = '二狗'

def show_name(name):
    print(name)

在 main.py 导入并使用

import info

print(info.name)  # 二狗
info.show_name('dog')  # dog

可以使用 as 定义别名，但是定义了别名后，原来的模块名就不能使用了

同时导入多个模块，使用 , 分隔

import math, time, random

:tiger: from 模块名称 import 变量/函数/类/* * 代表导入所有

该方式导入后可以直接使用，不用模块名进行引用

变量

from info import name
print(name)

函数

from info import show_name
show_name('dog')

当导入的多个模块中具有相同的内容，如果使用 from...import 的形式，则后导入的会覆盖先导入的，此时，如果想两个都使用，应该使用 import 方式进行导入，并使用 . 进行调用

举例：有三个文件 info.py my_info.py main.py。info.py 与 my_info.py 中有相同的变量和方法

info.py

name = '二狗'

def show_name():
    print(name)

my_info.py

name = 'dog'

def show_name():
    print(name)

main.py

from info import *
from my_info import *

show_name()  # dog

my_info 后导入的,所以使用 my_info 模块中的 show_name() 方法, 且 info 中的 show_name() 方法无法进行调用

如果想两个模块的方法都使用，应使用 import 方式进行导入

import info
import my_info

info.show_name()  # 二狗
my_info.show_name()  # dog

new_lst = []

8. 文件

8.1 概述

  • 存储在计算机的存储设备中的一组数据序列就是文件，不同类型的文件通过后缀名进行区分

文件通常分为两类

  1. 文本文件：由于编码格式（ utf-8 ，unicode）的不同，所占磁盘空间的字节数不同
  2. 二进制文件：没有统一的编码，文件直接由 0 或 1 组成，需要使用指定的软件才能打开

8.2 基本操作

==打开文件==

变量名 = open(filename, mode, encoding)，如果文件不存在会进行创建

==操作文件==

变量名.read() 变量名.write(str)

==关闭文件==

变量名.close()

操作示例

def my_write():
    # 打开(创建)文件
    file = open('a.txt', 'w', encoding='utf-8')
    # 操作文件
    file.write('伟大的中国梦')
    # 关闭
    file.close()


def my_read():
    # 打开(创建)文件
    file = open('a.txt', 'r', encoding='utf-8')
    # 操作文件
    s = file.read()  # <class 'str'> 伟大的中国梦

    # 关闭
    file.close()

8.3 文件的写入

文件在打开后，将会从磁盘读取到内存，操作文件是在内存中操作，文件关闭后，内存中的文件保存到磁盘

文件的打开模式 模式说明
r 以只读模式打开，文件指针在文件的开头，如果文件不存在，程序抛异常
rb 以只读模式打开二进制文件，如图片文件

fruits = ['apple', 'banana', 'pear']
counts = [10, 2, 4]

            print('10个苹果')
        case 'banana', 2:
            print('2个香蕉')
        case 'pear', 4:
            print('4个梨')
