为什么有 hashCode

HashCode 其实主要用在哈希结构里面，比如 HashTable HashMap HashSet，如果不用这些的话，提供这个方法可以快速计算散列值，快速定位到对象可能所在的位置，当出现 HashCode 相同时，代表出现哈希碰撞，这个时候可以通过 equals 来判断对象是否相同，减少了从头遍历进行 equals 当比较。减少查找成本。

  • 哈希值一样，但是两个对象不一定相等
  • 哈希值一样并且 equals 方法也返回 true 才认为相等
  • 如果哈希值都不一样，就认为不相等 --> 前提是 如果 a.equals(b) == true，那么 a.hashCode() 必须等于 b.hashCode()。

为什么重写 equals() 时必须重写 hashCode() 方法？
因为 equals 相等，那必须哈希值相同，不然会出现对象相等，但是在哈希表里面不同的桶里面！

异常

Exception 和 Error 有什么区别？
  • Exception 是程序本身可以处理的异常
    • Checked Exception
    • Unchecked Exception
  • Error 是程序本身无法处理的异常
⭐️Checked Exception 和 Unchecked Exception 有什么区别？
  • 受检查异常必须捕获或者抛出，除了 RuntimeException 及其子类，其他的 Exception 都是受检查异常
  • 不受检查异常不处理也能正常通过编译
你更倾向于使用 Checked Exception 还是 Unchecked Exception？

默认使用 Unchecked，必要时才用 Checked，因为这种异常一般是bug，对待bug最好是让他暴露出来而不是捕获，除非这个异常也是业务的一部分

⭐️异常使用有哪些需要注意的地方？
  • 不要把异常对象定义为静态成员变量，因为异常对象在创建时会记录调用栈。如果后续反复抛出同一个异常对象，打印出来的栈信息可能仍然是创建异常时的位置，而不是本次真实抛出的位置。所以每次手动抛异常时，应该 new 一个新的异常对象。
  • 抛出的异常信息要有意义，但是不要泄漏敏感信息
  • 异常类型应该更加具体(范围尽可能小)
  • 不要重复记录日志

泛型

什么是泛型？有什么作用？
  • 泛型能把类型当作参数传进去，代码在写的时候不用指定死一种类型，而是用的时候再指定
  • 类型安全，编译器能检查放入的类型对不对 --> 写安全
  • 减少强制类型转换，不用手动转化类型去使用了，编译器能自动转换 --> 读安全、方便
  • 提高代码复用性，比如一个类，既能用 String，也能用 Integer，不用为每个类型写一个类 --> 提高复用性
泛型的使用方式有哪几种？
  • 泛型类、泛型接口、泛型方法
  • 静态方法里面的泛型能用类上声明的泛型吗？？
什么是泛型擦除机制？为什么要擦除?
  • 编译期间擦除，运行期间泛型已经不存在了
  • 为了引入泛型但是不创建新的类
什么是桥方法？

为了继承泛型类时保证多态

泛型有哪些限制？为什么？

泛型的限制一般是由泛型擦除机制导致的。擦除为 Object 后无法进行类型判断

  • 只能声明不能实例化 T 类型变量。

    // 不能这么写
class Box<T>{
    public T create() {
    	  return new T(); // 因为擦除后是 Object，JVM不知道它是 String 还是 User，没办法强转，强转会报错
    }
}

    不同于

    List<String> list = new ArrayList<>();
list.add("a");

String ele = list.get(0);

    List<String> 这里类型擦除后变为 List，里面的 T 全变成 Object，但是 Object 里面实际存储的是 String 类型，所以擦除后，编译器加的强转是能成功的。
    但是第一个 new T() 假设能擦除变成了 new Object()，就没办法再转为 String 或者什么了，会直接报错，所以 Java 直接禁止 new T()

  • 泛型参数不能是基本类型。因为基本类型不是 Object 子类，应该用基本类型对应的引用类型代替。

  • 不能实例化泛型参数的数组。擦除后为 Object 后无法进行类型判断。

    class Box<T> {
    private T[] arr = new T[10]; // 编译错误
}

    因为数组创建时，JVM必须知道数组元素的真实类型。

    String[] arr = new String[10]; // String 类型没错

Object[] objArr = arr; // --> 这一步是允许的，所以如果 JVM 不知道数组元素的真实类型，下一个语句就能执行成功！！！非常危险了
objArr[0] = 123; // ArrayStoreException 真实类型是 String, 你放 Integer!不行

    所以如果允许的话

    // 擦除后
class Box {
    private Object[] arr = new Object[10];

    public Object[] getArr() {
        return arr;
    }
}

// 调用处
Box<String> box = new Box<>();
String[] arr = box.getArr(); // 变为 String[] arr = (String[]) box.getArr(); 强转会失败 Object[] 不能变为 String[]

  • 不能实例化泛型数组。

    // 编译错误
List<String>[] arr = new List<String>[10];
// 只能声明
List<String>[] arr; // 可以

  • 泛型无法使用 instanceof 对类型参数 T 做运行期判断；getClass() 在擦除后也无法区分不同泛型实参（如 List<String> 与 List<Integer> 均得到 List.class）。

  • 不能实现两个不同泛型参数的同一接口，擦除后多个父类的桥方法将冲突

  • 不能使用 static 修饰泛型变量

反射

获取 Class 对象的四种方式
  • 直接通过 类名.class 获取：可以拿到 Class 对象，通常不会触发类初始化，也就是不会执行 static 代码块。
  • 通过 Class.forName("类全限定名") 获取：默认会触发类加载，并且会触发类初始化。
  • 通过 对象.getClass() 获取：对象已经存在，说明这个类之前已经被加载并初始化过；getClass() 本身不会再次触发类初始化。
  • 通过 xxxClassLoader.loadClass("类全限定名") 获取：通过类加载器加载 Class 对象，默认只加载，不会触发类初始化。

类加载器，作用是把 .class 文件加载到 JVM 里面，变成 Class 对象。

什么是反射？
  • 程序运行时获取类信息并操作类或对象
  • 平时写的代码编译期就确定了类型之类的
  • 让代码更灵活，可以处理编译时未知的类型，成为很多框架和库的基石

浮点数表示

32 位浮点数 float 怎么表示呢？

1 位符号位 + 8 位指数位 + 23 位尾数，比如 25.125

整数部分：25 转为二进制是 11001
小数部分：0.125 转为二进制是 0.001，做法是一直乘以二，取整数部分，小数部分变为 0 就可以结束，或者截断

0.125  0
0.25   0
0.5    0
1.0    1

结果就是 0.001

合起来 11001.001，也就是 1.1001001×241.1001001 \times 2^41.1001001×24

  • 是正数，所以符号位为 0
  • 指数位是 4，然后加上 127 变为 10000011 (因为指数位没有符号，所以取的时候也会减去127)
  • 尾数就是 1001001 后面补零

结果就是

010000011100100100000000000000000
