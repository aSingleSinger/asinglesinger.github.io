Java 基础

Java 基础常见面试题总结(上)

https://javaguide.cn/java/basis/java-basic-questions-01.html

  • JVM 的定义、一次编译，随处可以运行的关键
  • JDK、JRE是什么？三个东西的关系？

JVM 是 Java虚拟机，是运行Java字节码的虚拟机。Java 程序通过编译器生成字节码文件，然后经过JVM解释器生成机器码进行执行。不同的系统(Linux、Windows、macOS)有不同的虚拟机实现，目的是让相同的字节码文件能在不同的机器上产生相同的结果。字节码和不同系统的JVM实现是Java语言一次编译、随处运行的关键所在。

JDK 是一个开发工具包，让开发者使用，用来创建和编译Java程序，它包含JRE以及编译器javac和一些其他工具，比如 javadoc,javap(反编译工具)。
JRE 是运行已编译Java程序所需要的环境，主要包含 JVM、Java基础类库。
整体来说

  • JDK
    • JRE
      • JVM 负责跑程序
      • Java 核心类库
    • Java开发工具(javac,javap,javadoc,jar)

不过从Java9之后引入了模块化，不再提供独立的 JRE 安装包

  • .class 文件就是字节码, 拿着 .class 就可以去不同机器上的JVM跑了，结果都是一样的。

  • 编译器, Java 程序(.java文件) 通过 javac 编译器，Groovy 程序(.groovy文件)通过 groovyc 编译器，Kotlin 程序(.kt文件)通过 kotlinc 编译器。最终都生成 .class 字节码文件，让语言和平台解耦

  • 我们平时常用的 HotSpot VM 是一种 JVM 实现，只要符合JVM规范，也可以开发属于自己的JVM

  • 字节码怎么得到
  • 用字节码的好处
  • 解释器&JIT

JVM 可以理解的代码就叫做字节码，通过 javac 编译器把 .java 源码编译后生成的 .class 文件，它不面向任何处理器，只面向虚拟机，所以Java程序无需重新编译便可以在不同操作系统的计算机上运行。

Java程序从源代码到运行的过程

.class 到机器码这一步。JVM会先加载字节码文件，然后通过解释器逐行解释执行，这种方法执行速度相对比较慢(同一个方法执行100次就要解释100次)。所以后来引入了 JIT 编译器，它属于运行时编译，JIT 编译器完成第一次编译后，就会把字节码对应的机器吗保存下来，下次可以直接使用。

  • JIT会监控代码运行，把热点代码编译成机器码。

关系图

  • Java不提供指针访问内存，程序内存更安全
  • C++提供多继承，Java是单继承，但是接口可以多实现
  • 他俩都能继续呢方法重载，Java不能进行操作符重载(因为操作符重载增加了复杂性，与Java最初的设计思想不符)
  • Java有自动内存管理 垃圾回收机制(GC), 不用手动释放内存。

标识符就是给变量、方法、类起名字，关键字就是背赋予特殊含义的标识符。

有点类似类和对象的关系

default 在三个地方都用到了，在 switch 里面来写默认的情况，在类里面，用这个来定义一个方法的默认实现，在访问控制里面，如果一个方法前没有任何修饰符，默认会有一个修饰符 default，但是如果手动加上会报错，因为只有(public/protected/private三种合法，不加默认是包私有权限)

++在前就是先加后赋值
++在后就是先赋值再加

  • >>>、>>、<< 区别
  • 移位的定义(把数据看作二进制数，然后左移右移)
  • 移位运算符更高效(一般一个时钟周期,位运算、算数加减也是一个时钟，乘法略长3-10，除法和取模20-80，所以尽可能用别的取代除法、取模)
  • 如果 int 移位超过 32 位如何？
  • 有哪种运算符不能移位吗？

比如 hash % 16 要十几个周期，hash & 15 只要一个周期

MQ概述 传统项目架构下的问题：请求方向响应方发送请求，如果中间链路出现问题，那么信息无法送到，请求方也没法发送了。 MQ全称 Message Queue，是在消息的传输过程中保存消息的容器。多用于分布式系统之间进行通信。 中间放一个MQ，生产者把消息给MQ，MQ把消息往下分发给消费者。 优势应用解耦消费方存活与否不影响生产方 比如如下情况 当用户买商品时，库存系统、支付系统、物流系统都会接收到这个消息产生对应动作，各个系统干各自的任务，完成业务解耦。 这时候，比如我想收到订单信息后再进行一个大数据统计，这个时候就可以单独做一个模块就可以，然后让MQ把消息发送给大数据系统就可以了。这个时候订单系统、其他系统都没有发生变化。 异步提速生产方发完消息，可以继续下一步业务逻辑 比如这里，原本如果按正常传统逻辑，需要先把数据存入数据库50ms，通知库存系统修改库存200ms，通知支付系统调用支付200ms，通知物流系统200ms，加起来有650ms，但是如果有MQ，我只需要先向数据库发送消息50ms，向MQ发送消息50ms，之后MQ将三分消息分别下发，这三个系统可以并行进行了...
