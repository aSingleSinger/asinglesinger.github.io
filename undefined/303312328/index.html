首先介绍一下什么是表达式？

在Java中，表达式（Expression）是由操作数（operands）和操作符（operators）以及方法调用和其他可以产生一个值的代码片段所组成的。==表达式通常会计算出一个值，并且可以作为语句的一部分、方法的参数或赋值语句的右侧==。

在Java中，表达式可以是以下类型的组合：

  1. 变量：使用变量名作为表达式，例如：int x = 10; 中的 x 是一个表达式。
  2. 常量：字面量可以作为表达式，例如：5, 3.14, 'a'等。
  3. 运算符表达式：由运算符连接的操作数，例如：x + y, a * b, !flag等。
  4. 方法调用：调用方法并获取返回值，例如：Math.sqrt(16), objectInstance.methodName()。
  5. 对象创建表达式：使用 new 关键字创建新的对象，例如：new ClassName()。
  6. Lambda表达式：用来表示函数式接口的实现，例如：(x, y) -> x + y。
  7. 数组访问表达式：用于访问数组中的元素，例如：arrayName[index]。
  8. 类型转换表达式：将一个类型的值转换为另一个类型，例如：(int) 3.14。
  9. 条件表达式（三元表达式）：根据条件选择不同的值，例如：condition ? value1 : value2。

而Lambda表达式的合法形式其实只有两种

第一种不带花括号的，主体需要是一个表达式作为返回值。

第二种主体带花括号的，如果主体不进行 return ，那么就是没有返回值。

所以该问题中的 process(() -> System.out.println("This is awesome")); 就出现了问题，因为 System.out.println() 是一个语句，而不是表达式（没有返回值，如果有返回值则可看作一个表达式）。所以理应是使用第二种带花括号的形式哦。剩下的就如书上的特殊规定。

System.out 是一个对象， println 是其中的一个方法。

3.5.4 使用局部变量

解释: Lambda可以没有限制的捕获（也就是在其主体中引用）实例变量和静态变量，但是局部变量必须显式声明为 final 或事实上是 final

在Java中，Lambda表达式可以捕获（引用）外部作用域中的实例变量和静态变量，而不需要显式地将它们声明为final或事实上是final。这是因为对于实例变量和静态变量，它们的生命周期通常比Lambda表达式更长，Lambda表达式只是捕获了对这些变量的引用，而不是变量本身。这种捕获方式在Java中是没有限制的。

相反，对于局部变量（在方法、构造函数或任何块作用域中声明的变量），Lambda表达式可以访问但必须是final或事实上是final的变量。"事实上是final"指的是变量虽然没有明确声明为final，但在其后面的代码中没有对其进行修改。这种限制是为了确保Lambda表达式中引用的局部变量在Lambda表达式执行时保持其原始值。如果局部变量不是final或事实上是final的，Java编译器将会产生编译错误。

    • Comparable 表示需要实现这个接口。

因此，U extends Comparable<? super U> 这个约束表明泛型类型 U 必须满足两个条件：

  1. U必须实现（或继承）Comparable接口。
  2. Comparable接口中的泛型类型必须是U的超类或U本身。

  2. 传入的参数必须是 U 或 U 的子类（其中 U 必须是可比较的，且可以和本类或者超类比较）。

语法糖是什么

“语法糖”（Syntactic Sugar）是指一种编程语言特性，它不会带来新的功能，但能够使代码更易读、更易写、更易理解，从而让程序员更高效地编写代码。这些特性使得编写代码变得更加简洁、清晰，但并没有引入新的概念或功能。

语法糖通常是针对某种模式、语法或常用的编码模式，提供更简洁、更易读的替代方式。它们并不会改变语言的基本结构或语义，而只是提供了更方便的编写方式。

一些常见的语法糖包括：

  1. For-each 循环：像 Java 中的 for-each 循环，它隐藏了迭代器或索引的复杂性，使得遍历数组或集合的代码更加简洁。
  2. 自动装箱和拆箱：在某些语言中（例如 Java），自动装箱和拆箱允许原始数据类型和其对应的包装类之间进行隐式转换，使得代码编写时更方便，不需要显式进行类型转换。
  3. Lambda 表达式：Lambda 表达式是一种更简洁地表示匿名函数的方式，可以用更少的代码来表示函数式接口的实现。
  4. 字符串插值：像 Python f-strings 或 JavaScript 的模板字符串，它们允许在字符串中直接插入变量或表达式，使得字符串的拼接更加清晰、简洁。
  5. 属性访问器：一些语言提供了属性访问器，允许通过类似属性访问的方式来访问对象的属性，而不必显式地调用 getter 和 setter 方法。

语法糖通常使代码更易读、易写、易懂，但有时也可能导致一些性能上的隐患，因为某些语法糖会引入额外的处理步骤。尽管如此，它们作为编程语言的一部分，对于提高开发效率和代码可读性有着重要的作用。

解释：无状态操作与有状态操作

无状态的操作是指流中的元素之间不会相互影响，如 map、filter 都是每个元素各自执行函数，互不影响。称为无状态操作，一般这种操作进行并行效果都不错，但是当在大数据量下，可能出现线程竞争或资源竞争，要进行性能测试并合理评估并行处理的影响后再决定。

有状态的操作就是要利用之前的信息。比如 skip、limit 都要记录已跳过个数或已留存的个数。

解释：无界与有界

无界：sorted distinct 在处理大规模甚至是无限的元素时，它们需要不断地维护状态空间，而这个状态空间并没有明确的上限。因此，这些操作被描述为无界操作，可能会对内存或性能产生潜在的影响。（后面元素的排序和去重依赖于前面的元素，所以需要所有元素都放入缓冲区和才能来计算是否向输出流中放入元素）

有界：reduce 在流处理过程中能够产生一个最终的结果，并不需要维护流中所有元素的状态。（处理一个就有一个结果，并不需要对元素信息保存）

6.5.1 理解 Collector 接口声明的方法

supplier() 在并发时会执行几次

在每个线程都会执行一次，这样做是为了保证并行操作的正确性和线程安全性。每个部分都有自己独立的结果容器，避免了多个线程在同一个结果容器上进行操作可能引发的并发问题。

然后，在并行操作的结束阶段，这些部分的结果会被合并成一个最终的结果。在这个阶段，这些结果容器将会被合并为一个单一的结果。

对于有序数据流，如果设置 Characteristics 为 CONCURRENT ，那么它会进行并行处理吗？

对于有序数据流，即使设置了 CONCURRENT 标志，通常情况下 Java Stream API 会尊重数据的有序性。这意味着它会尽可能保持数据的顺序，除非并行执行对结果顺序没有影响或者确保有序性会导致性能下降。

因此，CONCURRENT 标志并不会自动导致并行处理，而是表示该流可以安全地在并行环境中操作。Java Stream API 会根据多个因素来决定是否以及何时进行并行处理，并且对于有序数据，会尽可能地保持数据的有序性。

6.6.1 仅用质数做除数

延迟求值的特性（如 takeWhile ）是如何实现的

延迟求值在流处理中的实现通常基于惰性计算（Lazy Evaluation）的概念。底层实现可以通过对数据的封装和延迟操作来实现延迟求值。这种方式可以分为两个关键方面来解释：

  1. 封装数据结构：
    • 流操作通常是基于数据源（例如集合、数组或者生成器）进行处理的，但它并不会立即对整个数据源进行计算或操作。
    • 延迟求值的实现可以通过将数据源封装在特定的数据结构中，这些数据结构可以维护对原始数据的引用，同时记录下需要对数据进行的各种操作（如 filter、map、takeWhile 等）。
  2. 惰性操作：
    • 当对流应用操作时，实际的处理并不会立即发生，而是形成了一个操作的链条。每个操作仅记录了应该对数据执行的具体操作步骤，而不是立即对数据进行处理。
    • 当需要获取结果时（例如调用 forEach、collect 或者其他终端操作），才会触发实际的计算过程。这时才会按照操作链条中记录的顺序逐步处理数据并生成最终结果。

在实现延迟求值时，对流的各种操作（比如 filter、map、takeWhile 等）并不会立即处理整个数据集，而是将操作组合成一个操作链。当执行终端操作时，这个链条才会被触发并且按需执行每个操作，只处理必要的元素，并在满足条件后停止处理。
